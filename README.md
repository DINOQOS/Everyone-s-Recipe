# 모두의 레시피

**TEAM:** 한상민과아이들  
**DATE:** 2024-02-02  
**MEMBERS:** 한상민, 김윤성, 백성욱, 장정우

## 만든 이유

Spring Boot를 이용한 프로젝트로, 회원가입, 로그인, 게시물 작성, 추천/비추천 기능, 사진 업로드 등 다양한 기능을 구현할 수 있는 주제로 '나만의 요리 레시피 커뮤니티'를 선택했습니다.

## 개요

작업 순서는 다음과 같습니다:
1. **Entity 생성:** 데이터베이스 테이블을 생성하고, JPA Repository를 활용하여 MySQL Workbench와 상호작용을 하였습니다.
2. **DTO 생성:** DTO와 Entity 간의 변환을 위해 `toEntity`, `toDto` 함수를 구현하였습니다.
3. **View 생성:** JSP를 사용하여 사용자에게 입력받는 인터페이스를 구현했습니다.
4. **Controller 및 Service 구현:** `PostMapping`, `GetMapping` 어노테이션을 사용하여 요청을 Controller에 전달하고, Service에서 비즈니스 로직을 구현하였습니다.

## 느낀 점 및 어려웠던 점

### 한상민
1. JSP 파일을 학습한 후 Thymeleaf 템플릿 엔진을 사용해 보려 했으나 시간 제한 때문에 JSP 템플릿을 충분히 학습하고 적용하는 데 집중했습니다.
2. CKEditor4를 적용하여 사용자 친화적인 웹 에디터를 제공하려 했으나, 이미지 업로드 경로 문제를 해결하지 못했습니다. DB에 저장된 HTML 태그와 긴 텍스트를 처리하기 위해 컬럼 자료형을 LONGTEXT로 변환했습니다.
3. GitHub 협업 시 branch를 새로 만들어 conflict를 해결하며, 협업이 작업 효율성을 크게 높일 수 있음을 깨달았습니다.
4. DB에서 Nullable, default, 자료형 등의 사전 정의가 중요하다는 것을 알게 되었습니다.

### 김윤성
1. 초기에 3-layered 아키텍처 패턴을 이해하는 데 어려움을 겪었지만, 팀원과의 대화를 통해 개념을 정리했습니다.
2. 좋아요/싫어요 기능 구현 시 'like' 변수명 사용으로 인해 발생한 문제를 해결하면서 변수명의 중요성을 깨달았습니다.
3. Annotation에 대한 이해가 부족했으나, 팀원들의 도움으로 개념을 습득하고 주요 기능을 성공적으로 구현할 수 있었습니다.
4. GitHub 협업에서 개인 Branch를 활용하여 CONFLICT 문제를 해결하는 방법을 익혔습니다.
5. JSP에서 컨트롤러로 데이터를 넘길 때 혼란을 겪었으나, sysout을 사용한 디버깅과 @RequestParam의 활용으로 문제를 해결했습니다.

### 백성욱
1. Board와 관련된 객체들의 코드를 작성하며 복습을 통해 이해도를 높였습니다.
2. header.jsp를 수정하여 스크롤 시에도 항상 화면 상단에 고정되도록 구성했습니다.
3. 검색 기능을 구현하여 글 제목에 포함된 단어로 검색할 수 있도록 했습니다.
4. 회원 탈퇴 기능을 구현하여 탈퇴 시 작성했던 모든 글이 삭제되도록 구성했습니다.

### 장정우
많이 부족했지만, 팀원들과 함께 도우며 여러 기능들을 성공적으로 구현했습니다.

1. **User 기능 구현:** User와 Board로 나누어 진행했으며, User 기능을 먼저 구현했습니다. 이후 사용자가 음식을 분류할 수 있도록 image 데이터를 찾았습니다.
2. **Kaggle 데이터셋 활용:** 한식 데이터를 찾기 어려워 Kaggle의 Food-101 데이터셋을 활용했습니다.
3. **AI 모델 학습 및 배포:** Teachable Machine을 통해 프로토타입 모델을 배포하고, VGG-16 모델을 통해 48개의 클래스를 분류할 수 있도록 학습했습니다.
4. **모델 서빙:** Flask를 활용하여 API를 배포하고, JSP에서 AJAX를 활용하여 이미지 입력 기능을 구현했습니다.
5. **분류된 음식 검색:** 분류된 음식을 클릭하면 해당 음식 이름으로 게시판에서 검색이 이루어지도록 구현하여 다양한 레시피를 볼 수 있게 했습니다.

어려웠던 점:
- 한식 데이터 찾기가 어려웠습니다. 많이 사용되는 Food-101 데이터셋을 활용했으며, Roboflow의 한식 데이터는 사용에 어려움이 있어 다른 데이터로 진행하였습니다.
